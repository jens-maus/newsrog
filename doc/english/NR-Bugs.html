Known Bugs
Potentially Fatal Bugs While we have attempted to provide the highest quality software we can, there are always bugs in any complex application, and it is best if you know about them rather than stumble on them yourself. This is the exhaustive list of known bugs in NewsRog. We will be addressing them as soon as possible, particularly the fatal errors.

Please report new bugs to us: see Contact for more information.

Area Description Workaround
UUDecode Display Filter If an article is displayed using this filter, then the display filter object is deleted, and lastly, memory pressure forces an expunge of the library, it is possible for the system to crash. This should be a very rare combination of events however. Avoid

Display Glitches & Other Non-Fatal Bugs These bugs are non-fatal and are chiefly harmless display glitches of various types.

Object Class Description Status
Group Window If the group headers are saved while some articles are queued for network download, the queue icon falsely appears next time the group is loaded. NewsRog Bug
Group Window At times, articles displayed in the group window are unavailable from the news server. The expire option in the group window menu can be used to force an expire of old group articles, but at times even this leaves unavailable articles. NNTP Specification Issue
UUDecode Display Filter On screens of less than 256 colors, the datatypes display of inline images is often blank. Workaround: switch to a 256 or truecolor (16 or 24) bit display mode. NewsRog is designed to run optimally in at least 1024x768x16-bit. Datatypes bug.
